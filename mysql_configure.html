配置MySQL常见错误解决
发布于：2022-12-08 19:08:03 标签：/ MySQL / 访问：

前言

按教程安装并配置MySQL的过程中遇到了五花八门的报错，感觉算是把能踩的坑都踩完了，写一篇博客记录一下常见的报错及其解决方案。
使用.zip安装文件进行配置，参考教程：MySQL下载与安装

报错

Install/Remove of the Service Denied!

在输入mysqld --install后报错：Install/Remove of the Service Denied!

解决方法：
使用管理员权限运行cmd，并再次输入mysqld --install即可：

Access denied for user ‘root‘@’localhost’

大部分教程中都会提到“mysql初始密码为空，所以第一次输密码时直接按回车即可”，但我在安装时却遇到了报错：

解决方法：
出现这种情况是因为在前面使用mysqld --initialize --console进行初始化时已经生成了初始密码。
在控制台中找到A temporary password is generated for root@localhost: 字段，后面即为初始密码：

再次执行mysql -u root -p后输入上述密码即可成功登录：

如果已经关闭了控制台，请在根目录下的data文件夹里找到后缀名为err的文件，打开并找到A temporary password is generated for root@localhost: 字段，后面即为初始密码。

参考教程

failed to set datadir to XXX

解决方法：
输入mysqld --initialize --user=mysql --console
随后输入mysqld -install
如果出现The service already exits!的提示则输入sc delete mysql，并再次输入mysqld -install

参考教程

解决方法：

mysqld --remove→mysqld --install→net start mysql即可解决
